# TungsAcu-DB 架構重設計 — 2026-04-27

## 核心原則

資料與程式分離：後端為人類可直接編輯的 CSV 檔，前端 Streamlit 只負責讀取與呈現。Claude 負責從 OCR 文字提取資料填入 CSV，人工審閱後即可上線。

---

## 後端架構

### acupoints.csv — 穴位表

一行一個穴位，資料來源限定：《董氏奇穴穴位詮釋解》（楊維傑，人民衛生出版社 2018）

| 欄位 | 說明 | 格式規範 |
|------|------|---------|
| 穴名 | 穴位全名 | 例：靈骨穴 |
| 部位 | 董氏分部 | 一一、二二…增補 |
| 取穴 | 位置描述 + 現代解剖 | 自由文字 |
| 主治關鍵字 | 臨床術語，逗號分隔 | 每個詞以 6 字以內為原則，一詞一症 |
| 針法 | 針刺方向、深度 | 例：直刺 1–2 寸 |
| 理論摘要 | 董楊核心概念 | 100 字以內 |
| 圖片路徑 | 對應圖檔相對路徑 | 例：images/靈骨穴.png |
| 詳細筆記路徑 | 對應 .md 筆記相對路徑 | 例：notes/靈骨穴.md |
| 備註 | 其他補充（含艾灸需求） | 自由文字 |

### pairs.csv — 對針 / 穴組表

一行一個組合（2 個以上穴位），資料來源：楊維傑各著作

| 欄位 | 說明 | 格式規範 |
|------|------|---------|
| 名稱 | 組合慣用名（可空白） | 例：靈骨大白 |
| 包含穴位 | 逗號分隔 | 例：靈骨穴,大白穴 |
| 適用症狀 | 逗號分隔關鍵字 | 同主治關鍵字規範 |
| 優先順序 | 整數，越小越優先 | 1 = 最重要，預設 99 |
| 來源書籍 | 出處 | 縮寫即可 |
| 備註 | | 自由文字 |

### notes/ — 穴位詳細筆記

每個穴位一份 `.md` 檔，存放完整董楊原文、理論推導、臨床心得。CSV 的「詳細筆記路徑」欄位指向此檔。

## 前端架構（Streamlit）

### 三個查詢入口

**① 穴位查詢**
- 搜尋穴位名稱
- 顯示：取穴、主治關鍵字、理論摘要、圖片
- 旁邊放「📖 詳細」連結，開啟對應 `.md` 筆記

**② 症狀查詢**（核心功能）
- 輸入症狀關鍵字（如「頭痛」）
- 顯示兩區塊：
  - **單穴推薦**：主治關鍵字含此詞的穴位清單
  - **對針 / 穴組推薦**：適用症狀含此詞的組合，按優先順序排列
- 點選組合內的穴位名稱 → 跳至穴位詳情頁

**③ 對針瀏覽**
- 列出所有對針組合
- 可按部位或症狀篩選

### UI 設計參考（tung-acupoints）

- **左右分割面板**：左側清單，右側詳情，比 tab 式更直覺
- **部位篩選按鈕**：一一、二二…等各部位直接用按鈕切換，不用下拉選單
- **傳統美學風格**：米色底、朱紅、金色配色（可參考採用）

## 資料填入流程

```
桌機 MinerU hybrid（分批，每次 50-100 頁）
  → 輸出 .md 檔
  → Synology Drive 同步到筆電
  → Claude 讀 .md 按欄位規範填入 CSV
  → 人工用 Excel/Numbers 審閱校正
  → git commit + push → Streamlit Cloud 自動更新
```

**MinerU 注意事項**：全書 400 頁需分批跑，每批 50-100 頁，在桌機 tmux session 內執行。

## 遷移策略

現有 SQLite 資料（196 穴，dongzhen_new.db + dongshi.db）不丟棄：
1. 從 SQLite 匯出現有欄位 → 填入 acupoints.csv（作為初始值）
2. 對針資料從 dongshi.db 匯出 → 填入 pairs.csv
3. 缺失欄位（理論摘要、取穴解剖等）由 Claude 重新從 OCR .md 提取補齊
4. 確認 CSV 資料完整後，SQLite 退役
