# 穴位主治關鍵字化與對針排序 Spec — 2026-04-26

## 望：現狀觀察

- `dongzhen_new.db` 的 `acupoints` 表共 196 穴，`dong_indications` 欄位存放從《董氏奇穴穴位詮釋解》OCR 直接抽取的書本原文段落（如「治頭部諸症，額頭痛甚效，並治三叉神經炎」）
- `詮解發揮` 相關欄位（`commentary`、`name_explanation`、`comparison_text` 等）填充率極低（80–99% 空白）
- `dongshi.db` 存有對針組合（`acupoint_pairs`）與症狀主治（`symptom_treatments`），但「對針組合」tab 目前按書名分組，無頻次排序
- `app.py` 的穴位詳情頁直接顯示 `dong_indications` 原文，無轉換層
- 管理員編輯介面已存在，支援直接寫回 SQLite

## 聞：需求陳述

TungsAcu-DB 資料庫內容太破碎，穴位主治都是書中截取內容，品質很低，文字破碎導致在臨床上不好用。兩個段落（董師原文和詮解發揮）都有問題。

希望查穴時能看到乾淨的主治關鍵字（例如頭痛、肩膀痛、鼻過敏），並列出對針或常用穴組，按照常用度排序。先做小範圍驗證。

## 問：澄清記錄

- **範圍**：首批選最常用的 20–30 穴，不一次全做
- **目標格式**：關鍵字在前、原文括號在後，例如「頭痛、肩膀痛（治頭部諸症，額頭痛甚效）」
- **關鍵字來源**：AI（Claude API）批次產出，人工管理員介面審閱確認
- **常用度定義**：書中出現次數為主，未來視需要再加入手動排序功能
- **範圍外**：詮解發揮欄位補全、對針資料庫品質審計（本次不處理）

## 切：技術脈絡

- **資料庫**：SQLite，`dongzhen_new.db`（主庫）＋ `dongshi.db`（補充庫）
- **新增欄位**：用 `ALTER TABLE` 加欄，不重建資料庫，不影響現有資料
- **Claude API**：筆電端（samue11in）可呼叫，用來批次精煉 `dong_indications` → 關鍵字清單
- **顯示邏輯**：`app.py` 有 `@st.cache_data`，修改欄位後需 `load_acupoint.clear()` 清快取
- **Fallback 約束**：精煉未完成的穴位必須繼續正常顯示，不能出現空白或例外
- **部署**：git push → Streamlit Cloud 自動更新，DB 隨程式碼一起 commit

## 病因病機

### 系統模型分析

- **脾（資料加工）**：`parse_dongzhen.py` 只搬運 OCR 文字到欄位，缺少語意精煉步驟
- **腎（儲存）**：`dong_indications` 存書本散文，非結構化臨床術語
- **肺（顯示）**：`app.py` 無轉換層，原文直接呈現
- **經絡（對針通道）**：`dongshi.db` 對針排序無頻次依據，臨床最常用的組合不會優先出現

### 病理模式

**脾虛夾痰**——建庫管道缺乏語意精煉能力（脾虛），導致未消化的書本原文直接堆積在資料庫（成痰），阻礙後續臨床查詢。

### 根因推演

由於建庫管道缺少「原文 → 臨床關鍵字」的精煉步驟，導致 `dong_indications` 儲存的是書本散文而非可用術語，最終表現為醫師查穴時需要自行閱讀解析原文，無法快速判讀。

## 治療設計

### 治法方向

**補法 + 調和**：在資料加工層（脾）補入語意精煉步驟，並調和對針顯示的排序邏輯。

### 具體方案

**① 新增 `indications_kw` 欄位**
```sql
ALTER TABLE acupoints ADD COLUMN indications_kw TEXT;
```
保留原始 `dong_indications` 不動，新欄位存精煉後關鍵字。

**② 建立 `extract_keywords.py` 精煉腳本**
- 輸入：`dong_indications` 原文
- 呼叫 Claude API，prompt 要求輸出「3–8 個繁體中文臨床關鍵字，逗號分隔」
- 輸出寫入 `indications_kw`
- 首批穴組與單穴如下（依 dongshi.db 出現頻次確認）：

**穴組（共用同一份關鍵字）**

| 穴組 | 包含穴位 |
|------|---------|
| 婦科組 | 婦科穴、還巢穴 |
| 三重組 | 一重穴、二重穴、三重穴 |
| 重子重仙組 | 重子穴、重仙穴 |
| 天地人皇組 | 天皇穴、地皇穴、人皇穴 |
| 正筋正宗組 | 正筋穴、正宗穴、正士穴 |
| 四花組 | 四花上穴、四花中穴、四花外穴 |
| 水通水金組 | 水通穴、水金穴 |
| 側三里組 | 側三里穴、側下三里穴 |

**單穴**：靈骨穴、大白穴、下白穴、火主穴、水相穴、通山穴、火菊穴、火山穴、曲陵穴、木留穴、火硬穴、天皇副穴（腎關穴）

**③ app.py 顯示邏輯更新**
- 「董師原文」tab 的主治欄：若 `indications_kw` 非空，顯示「{kw}（{dong_indications}）」；否則 fallback 顯示 `dong_indications`
- 修改 `load_acupoint` 查詢，SELECT 加入 `indications_kw`

**④ 對針頻次排序**
- 在 `load_pairs_for_acupoint()` 查詢加上：
  ```sql
  GROUP BY point1, point2
  ORDER BY COUNT(*) DESC, source
  ```
- 顯示時附上出現次數標籤（如「× 3 本」）

**⑤ 管理員編輯介面新增 `indications_kw` 欄位**
- 加入現有 `edit_fields` 清單，label「主治關鍵字（精煉版）」

## 預期結果

1. **關鍵字覆蓋率**：首批選定的 20–30 穴，`indications_kw` 欄位 100% 填入，每穴包含 3–8 個乾淨的臨床關鍵字（不含完整書本句子）

2. **顯示格式正確**：app 顯示時，有 `indications_kw` 的穴位呈現「關鍵字（原文）」格式；尚未精煉的穴位 fallback 顯示原文，不出現空白或錯誤

3. **對針頻次排序**：穴位詳情頁「對針組合」tab 的結果依書中出現次數降序排列，最高頻的組合出現在最上方，可目視驗證

4. **管理員可編輯**：管理員登入後，`indications_kw` 出現在編輯 tab，儲存後即時反映在前端

5. **不破壞現有資料**：`dong_indications` 原欄位內容不變，現有功能（搜尋、部位瀏覽）維持正常
